# Smart Campus Management System

A full-stack campus management platform featuring **AI-automated attendance** via facial recognition, **constraint-based class scheduling**, **centralized dashboards**, and **automated email alerts**.

## 🏗️ Architecture

```
┌──────────────┐       REST API (JWT)       ┌──────────────────┐
│   React.js   │ ◄──────────────────────── │  Django + DRF    │
│   (Vite)     │                            │  Python Backend  │
│   Port 5173  │                            │  Port 8000       │
└──────────────┘                            └────────┬─────────┘
                                                     │
                                            ┌────────▼─────────┐
                                            │   PostgreSQL     │
                                            │   (or SQLite)    │
                                            └────────┬─────────┘
                                                     │
                                            ┌────────▼─────────┐
                                            │   Face Recog.    │
                                            │   Engine (OpenCV │
                                            │   + dlib CNN)    │
                                            └──────────────────┘
```

## 🚀 Quick Start

### Prerequisites

- **Python 3.10+**
- **Node.js 18+**
- **PostgreSQL** (optional — SQLite works out of the box)
- **CMake + C++ compiler** (for face_recognition library)

### 1. Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Copy environment config
copy .env.example .env       # Windows
# cp .env.example .env       # macOS/Linux

# Run migrations (creates SQLite database by default)
python manage.py makemigrations accounts attendance scheduler communication
python manage.py migrate

# Create superuser (optional, seed script creates one)
# python manage.py createsuperuser

# Seed demo data
python seed_data.py

# Start the server
python manage.py runserver
```

The backend will be running at **http://localhost:8000**

### 2. Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start dev server
npm run dev
```

The frontend will be running at **http://localhost:5173**

### 3. Open in Browser

Navigate to **http://localhost:5173** and use these demo credentials:

## 📦 Modules

### 1. AI-Automated Attendance (`face_recognition_engine/`)

Uses **dlib's CNN model** (via `face_recognition` library) + **OpenCV** for real-time face detection.

**Setup:**
```bash
# Organize student photos by enrollment number:
backend/face_recognition_engine/training_images/
├── STU001/
│   ├── photo1.jpg
│   └── photo2.jpg
├── STU002/
│   └── photo1.jpg

# Generate face encodings
cd backend
python face_recognition_engine/encode_faces.py

# Start live recognition (requires webcam)
python face_recognition_engine/recognize_faces.py --subject-id 1
```

**How it works:**
1. `encode_faces.py` scans training images, generates 128-d face vectors, stores in DB
2. `recognize_faces.py` captures webcam frames, compares faces against stored encodings
3. On match → marks attendance via REST API (`POST /api/attendance/mark/`)

### 2. Class Scheduler (`apps/scheduler/csp_solver.py`)

**Algorithm:** Backtracking CSP with MRV heuristic + Forward Checking

**Constraints:**
- No faculty teaches 2 classes at the same time
- No room hosts 2 classes at the same time
- Room capacity ≥ subject's required capacity
- Same subject not scheduled twice on the same day

**Usage:**
```bash
# Via API (admin only)
POST http://localhost:8000/api/scheduler/generate/

# Or via the admin dashboard "Generate Timetable" button
```

### 3. Centralized Dashboards

- **Admin Dashboard:** Stats overview, attendance charts, quick actions, recent notices
- **Student Dashboard:** Personal attendance %, today's schedule, notices
- **Timetable View:** Color-coded weekly grid generated by CSP solver
- **Attendance Records:** Filterable data table with AI vs manual marking indicators

### 4. Communication & Alerts

- **Digital Notice Board:** Priority-coded (low/medium/high/urgent) announcements
- **SMTP Email Automation:** Attendance shortage alerts + notice notifications
- **Trigger alerts:** `POST /api/communication/alerts/attendance/`

## ⚙️ SMTP Email Configuration

To enable real email sending (instead of console output):

1. **Gmail Setup:**
   - Enable 2FA on your Google account
   - Generate an App Password: Google Account → Security → App Passwords
   - Update `.env`:
     ```
     EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
     EMAIL_HOST=smtp.gmail.com
     EMAIL_PORT=587
     EMAIL_USE_TLS=True
     EMAIL_HOST_USER=your-email@gmail.com
     EMAIL_HOST_PASSWORD=your-16-char-app-password
     ```

2. **Other Providers:**
   - **Outlook:** `EMAIL_HOST=smtp-mail.outlook.com`
   - **SendGrid:** `EMAIL_HOST=smtp.sendgrid.net`, `EMAIL_PORT=465`
